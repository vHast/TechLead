Bash and the Command Line

  • It is very probable that you may have heard of the Bash Shell if you've taken interest in getting certified in the CompTIA Linux+ XKO-004 Certification.
  • For beginners that don't know what a shell is, it's basically a computer program that presents a command line interface which allows you to control your computer using commands entered with a keyboard instead of controlling graphical user interfaces(GUIs).
  • Bash is simply the default interpreter on many GNU/Linux systems, before continuing we'd like you to invite to install any Linux distribution to get started using Bash, Mint works wonders for beginners.
  • Even if you're not interested in server administration, it's useful for upstarting your productivity once you get the hang of it.
  • To start your Bash terminal, locate the icon, or simply press Ctrl + Alt + T.
  • Your first task will be to determine which is your default interpreter
    • ~$ echo $SHELL /bin/bash
  • This is a good practice since we need to determine which our default interpreter is, as there are several other shell interpreters available, such as the C shell.

    • Navigation

  • In order to navigate through our folders, we are going to need to learn a few commands.
  • The important ones are pwd, ls, and cd.
  • pwd will print your current location.
    • ~$ pwd /home/(yourUserName)
  • ls will list the files in your current location, you can also add -a after ls in order to see the hidden files.
    • ~$ ls Documents Music Templates Desktop Downloads Pictures Public Videos
  • In order to access any of these locations, all you have to do is type cd and the folder you're interested in accessing.
    • ~$ cd Documents ~/Documents $
  • Once inside the folder you are, you can always go back with cd ..
    • ~/Documents $ cd .. ~$
  • This way of accessing your data is much faster than going around clicking icons and searching for the right folder to go.
  • There are tools that we could install into our system that will enhance the experience, one of my favorites is Vim which is a very good text editor, imagine that we have a text file filled with code.
  • Most of us would just launch Visual Studio Code and configure it, but if we know exactly what we want to do, we could just locate the file we want to edit through the command line and open it with Vim / Nano or Cat
    • ~$ ls Documents Music Templates Desktop ~$ cd Documents ~/Documents $ ls index.html ~/Documents $ vim index.html
  • First we check which documents are in our root with ls, after that we decide to go to Documents using the cd command.
  • We check again the contents of the folder with ls and we see that we've got in sight the file that we are interested in which is index.html
  • We finally open the document using the program of our interested with the name of the file we are interested in working in after
  • This method works for a lot of programs but some have different parameters, and you should check the documentation.
  • For example, I'm a big Doom fan and in order to play custom WADs I need to put -file after the program
    • ~$ zdoom -file btsx_e1_std.wad
  • It's a different nomenclature but the structure is similar for a lot of programs.
